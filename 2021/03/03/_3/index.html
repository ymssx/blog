深度学习 骨架检测 综述

# 基于传统方法

计算图像中梯度响应强弱 以及 边缘与骨架之间位置关系

# 基于边缘强度图的方法

边缘强度图本身对局部纹理变化和干扰十分敏感，只在干净背景图片以及前景纹理少的图片中有效。难以应用。

# 基于机器学习方法

人工设计特征并不能很好的检测复杂自然图片

Tsogkas：基于学习的边缘检测方法 -> 计算图像在颜色和纹理亮度上的差异性特征 => 多实例学习（MIL：multi instance learning）检测自然图像中的物体对称轴。

# 基于深度学习

HED图像轮廓检测：论文认为CNN在传播时，从底层到高层会学到不同层次的信息，底层：局部纹理特征，中层：形状、线条特征，深层：物体、类别等抽象特征。而轮廓检测需要同时兼顾底层纹理以及高层语义（帮助区分前背景）。

=> 因此需要利用侧边输出side-output

HED(Holistically-nested Edge Detection)以VGG16作为基础，在每一个池化层之前的卷积层（conv1-2，conv2-2，conv3-3，conv4-3，conv5-3）进行侧边输出。五个侧边输出的感受野分别是4、14、40、92、196。每个分支都是使用的同样的轮廓GT分别监督。

对于轮廓检测而言，感受野不影响轮廓的提取；但是对于骨架检测而言，没有足够的感受野是无法知道某个像素是否是骨架的 --- 不识庐山真面目，只缘身在此山中。

FSDS： 因此直接使用HED会使浅层的侧边输出无法检测到部分尺度的骨架，此时统一使用完全体GT是不合适的，SSO（scale-associated side-output）根据侧边输出的感受野来动态确定其感受野（将完整感受野拆分成不同尺度的几部分），侧边输出改为SSO

  1. 去掉侧边输出1（最浅的，感受野太小，没有价值）
  2. 每一个SSO输出多个尺度相关的骨架图，同一个尺度骨架图融合时使用权重加权
  3. 每一个SSO使用一个回归分支来预测骨架尺度

SRN：侧边输出side-output在不同层有不同的感受野和不同尺度语义信息，SRN从深到浅将侧边输出的特征图进行向下连接，使高层语义信息对底层进行增强。能够很好的检测多尺度的骨架。

Hi-Fi：同时融合高层与底层的信息，相互进行增强。

DeepFlux：不再将骨架预测当作一个逐像素二分类问题，而是预测一个二维向量场（two-dimensional vector field），将每一个scene指向一个候选的骨架像素（将每一个像素与附近的骨架点联系起来） => 基于通量(flux)的骨架算法

膨胀骨架

结构：HED + ASPP

优点：检测的是基于区域的向量场，其可以更好的处理大宽度的目标区域

ProMask：1. 使用高斯掩膜对骨架标签做处理，使得骨架标签平滑变粗 2. 使用vector router模块来帮助解决网络对空间位置敏感的问题，解决了骨架的连接问题。
